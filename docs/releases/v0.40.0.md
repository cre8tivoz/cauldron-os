# Cauldron OS v0.40.0

Cauldron OS v0.40 adds stronger review loops around both the blueprint and prototype, makes build-agent handoffs genuinely multi-agent, and opens the Taste Engine to community references.

## Highlights

### Review blueprint changes before prototype generation

Blueprint generation now keeps a capped version history. Restore an earlier snapshot or inspect a structured line-level diff between revisions before committing to prototype generation.

### Create scoped multi-agent handoffs

Select multiple detected build agents and generate a shared orchestration manifest with scoped packages under `agents/<agentId>/`. Single-agent and handoff-only workflows remain supported.

### Import community design references

The Taste Engine now includes a Community tab with curated DESIGN.md systems and scaffold starter guidance. Imported design systems are stored locally under runtime data, registered immediately, and discovered again on restart.

### See prototype quality before critique

Every generated prototype receives a deterministic local quality pass across:

- Accessibility
- Visual hierarchy
- Spacing
- Color contrast
- Semantic HTML

The Prototype stage shows an A/B/C/D grade. C and D output also receives focused PM-style suggestions before the natural-language critique box.

## Upgrade

```bash
git pull origin main
npm install
npm test
npm start
```

Open `http://localhost:3000`.

## Compatibility And Data

- Runtime data remains local under `data/` unless `CAULDRON_DATA_DIR` is configured.
- Existing drafts and generated projects remain compatible.
- Community imports are runtime data and are intentionally not committed.
- Existing single-agent handoff and scaffold export contracts remain supported.

## Validation

The release smoke suite covers core routes, cloud model routing, frontend contracts, design systems, community imports, multi-agent handoffs, scaffold generation/export, prototype quality scoring, critique iteration, and end-to-end handoff.

Release QA also includes local-only Playwright screenshots and a walkthrough recording stored under the ignored `test-images-codex/` directory.

## Known Follow-Ups

- Community scaffold starters currently add guidance on top of supported base scaffolds.
- Community catalog entries use a curated built-in fallback while the external community repository matures.
- Prototype quality scoring is heuristic and should support, not replace, human review.
